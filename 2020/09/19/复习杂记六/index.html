网络体系结构

(1) 典型网络
(2) 协议
(3) 网络设备
(4) 考前翻看

计算机网络是一个互联的 自治的系统

  • 功能

    数据通信

    资源共享

    分布式处理 Hadoop

    提高可靠性

    负载均衡

  • 组成部分: 硬件+软件+协议
  • 工作方式 : 边缘用户+核心部分
  • 功能组成 : 资源子网(数据通信)上四层+ 通信子网第三层(资源共享/数据处理)

    七层结构

  • 分类

    分布范围 : wan ,man (技术: 交换技术+广播技术)

    用户:公用网+ 专用网

    交换技术:电路交换+报文交换+分组交换

    拓扑结构: 总线型,星型, 环形,网状型

    传输技术 : 广播式+ 点对点(分组存储转发+路由选择)

    3.性能指标 (1)

  • 速率

    数据率/数据传输率/比特率

    速率: 用十进制表示

    存储容量: 用二进制表示

    带宽: 可以指频带宽度或指传输最高数据率, 计网中通常用第二个定义

  • 吞吐量

    单位时间欸通过某个网络(信道/网络设备)的数据量, 单位b/s,kb/s,Mb/s

    发送速率, 接受速率(受缓存等的影响)

  • 时延

    单位:秒

    从网络一端到另一端所需的时间

    发送时延+传播时延+排队时延+处理时延

    发送时延(将数据推送到信道上的时间)= 数据长度/信道宽度(发送速率)

    传播时延=信道长度/电磁波在信道上的传播速率

    排队时延(等待输出/入链路可用)

    时延带宽积(这段链路中有多少比特)=传播时延x带宽

    RTT: 从发送方发送数据开始,到发送方接受到确认, 一共经历的时延(接受方接受到的数据后立即放送数据).

    RTT = 往返传播时延+ 末端处理时间

    不包括发送时延

  • 利用率

    信道利用率 = 有数据通过时间/(有+无)数据通过时间

    网络利用率 : 信道利用率加权平均

    利用率增大,时延急剧增大

4.分层结构 (2)

  • 原因

    (1)对数据通路激活

    (2)告诉网络识别目的主机

    (3)查明目的主机是否接入网站

    (4)查明目的主机文件管理程序是否就绪

    (5)确保差错与意外可以解决

    …..
    利用分层将大问题简化

  • 手段

    实体->协议(一系列规则 怎么打包,怎么拆包 )->接口(两个层次间的通道)->服务(两层之间的关系, 上层要求服务,下层提供服务)

    原则一: 各层独立

    原则二: 界面自然清晰

    原则三:每层用最合适的技术

    原则四:保持下层对上层的独立性

    原则五: 整个结构有利于标准化工作

    实体: 弟n层活动元素称为n层实体

    协议:为对等实体数据交换而建立的规则

    协议语法: 规定数据格式

    协议语义: 规定要完成的工作

    协议同步: 规定各种操作的顺序

    接口(访问服务点ASP):上层使用下层服务的入口

    服务: 下层为相邻上层提供的功能调用

    SDU:为完成用户要求功能的数据

    PCI:控制协议操作的信息

    PDU: 对等层次之前传送的数据单位

    4.OSI参考模型(理论成功,市场失败)TCP/IP赛高 (1)

  • 结构 物联网淑慧试用

    1,2,3通讯子网

    5,6,7资源子网

    A (7层)– 中间系统(3层) – B(7层)

    中间层拆包主要是为了清楚传输信息

  • 应用层

    所有能和用户产生网络流量的程序

    FTP, SMTP, HTTP

  • 表示层

    两个通信系统中交换信息的方式(语法和语义)

    数据格式变换(翻译官)

    ..0100001001…->图片

    数据加密与解密, qq交流

    数据压缩与恢复, qq视频(JPEG, ASCII)

  • 会话层

    向表示层的实体/用户进程提供建立连接上有序地传输数据

    建立同步
    网页对用户操作建立同步, 会话间彼此独立

    建立,管理,终止会话

    使用校验点可使通信失效式,从校验点/同步点继续恢复通信,实现数据同步.断点续传, 适用于传输大文件(ADSP,ASP)

  • 传输层

    负责两进程的通信, 端到端通信

    可差流用

    单位: 报文段, 用户数据段.

    资源子网到传输子网的接口

    以下的层都是点到点的

    可靠传输(大数据,建立连接, 确认数据),不可靠传输(小文件)

    差错控制: 报文丢了,错了,进行纠正

    流量控制:控制发送方速度, 协调速度不匹配

    复用与分用:

    依据端口号

    复用:多个进程同时使用运输层服务

    分用:运输层将数据分开交付

  • 网络层 : 把分组从源端传到目的端

    单位: 数据报

    1.路由选择: 根据路由算法计算出最佳路径

    2.流量控制:限制发送方

    3.差错控制(奇偶校验码)

    4.拥塞控制:面向全局

    拥塞: 若所有节点都来不及接受分组,而要大量丢弃分组,则网络处于拥塞状态

    协议 IP,IPX ,IGMP,ARP,RARP

  • 数据链路层

    1.成帧: 定义帧的开始和结束

    2.差错控制 : 帧错+位错
    3.流量控制(防止接受方缓存不够用)

    4.访问(接入)控制: 控制对信道的访问

    协议: SDLC,HDLC,PPP,STP

  • 物理层

    作用:物理媒体上实现比特流的透明传输

    单位:比特

    透明传输

    1.定义接口特性

    2.定义传输模式:单工,半双工,双工

    3.定义传输速率

    4.比特同步

    5.比特编码

    5.TCP/IP五层参考模型 (1)

  • TCP/IP协议 栈

    应用层: HTTP,FTP,DNS

    传输层: TCP,UDP

    网络层: IP

    网络接口层: Ethernet,ATM ,Frame Relaw

  • 与OSI的相同点

    1.都分层

    2.都基于独立协议栈的概念

    3.可以实现异构网络互连

  • 不同点

    TCP/IP定义三点: 服务,协议,接口

    TCP/IP网络层: 无连接

    TCP/IP传输层: 无连接+面向连接

    OSI网络层: 无连接+连接层

    OSI传输层: 面向连接

    面向连接: 三个阶段(建立连接,传输数据,断开连接)

    无连接: 传就是了!!!

    TCP/IP关注异构网络: 所以把IP作为重要层次

  • 综合OSI与TCP/IP

    应用层: 支持各种网络应用 FTP SMTP HTTP

    传输层: 进程与进程通信 TCP UDP

    网络层 : 源主机到目的主机的数据分组路由与转发 IP ICMP OSPF

    数据链路层: 将网络层的数据包封装成帧 Ethernet, PPP

    物理层: 实现比特流的透明传输
